05b · Circuit Grid — Fundo Animado
Dois componentes: HeroBackground (hero, seção cheia) e CircuitBg (dentro de cards de foto). Grid ortogonal de nós + arestas horizontais/verticais com sinais viajando. Nunca diagonais.
HeroBackground
  • • Grid 10×4, step 110px
  • • viewBox 1100×440
  • • Ghost lines: rgba(59,130,246,0.13)
  • • Signals: 8 edges, dur 2.6–3.8s
  • • Glow blob azul, opacity 0.07
  • • Sem -z-10 (pai relative sem z-index)
CircuitBg variant="section"
  • • Mesma geometria do hero
  • • Usar em: FinalCTA section
  • • Linhas azuis sobre cream
  • • opacity global: 0.65
CircuitBg variant="card"
  • • Grid 7×5, step 60px
  • • viewBox 480×360
  • • Ghost: rgba(255,255,255,0.10)
  • • Signal: rgba(147,197,253,0.90)
  • • Usar dentro de photo cards escuros
Animações CSS: signal-travel — strokeDashoffset 120→-12, opacity 0→1→0 (edge traversal). node-glow — opacity 0.20→0.90 ciclo. node-ring — r expande com fade. hero-glow-drift — translate + scale em 20–26s. Regra: todas as coordenadas pre-computadas como constantes fora do componente para evitar hydration mismatch.

07 · Regras de Uso
✓ Fazer
  • • Parabole em palavras-chave de títulos
  • • DM Sans para corpo e subtítulos
  • • Foto + grain = card contido, não fundo
  • • Botões uppercase + ArrowUpRight
  • • Alternância base ↔ elevated entre seções
  • • text-wrap: pretty em todos headings
  • • Nunca palavra solitária na última linha
  • • Circuit grid só horizontal/vertical — nunca diagonal
  • • Coordenadas SVG pre-computadas fora do componente
  • • Video testimonial em terminal escuro (iframe 16:9)
✗ Não Fazer
  • • Parabole em texto corrido/body
  • • Foto como background de seção inteira
  • • Dark overlay pesado sobre fotos
  • • Texto em cima de foto (só UI/diagrama)
  • • Duas seções consecutivas com mesmo bg
  • • H1 acima de 4rem (64px)
  • • Amoeba blobs no tema light
  • • Diagonais no circuit grid
  • • Math.sin/cos dentro de render (hydration mismatch)
